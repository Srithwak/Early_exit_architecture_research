b)​ (Monique) 
The ML problem your project will address 
●​ Energy Use in Neural Networks: Application for Seizure Detection 
The core idea behind our machine learning problem is that traditional neural networks process each data 
layer in full. This takes unnecessary time, energy, and memory. Many neural network results can be 
derived from minimal or more targeted computation. Our solution proposes early-exit neural networks 
that present results as soon as the model is confident enough in its computation, saving energy while 
maintaining accuracy. Our project will apply this solution to seizure data and research to enable 
healthcare predictions.  
c)​ (200 words max) (Rithwak) 
Your project goal and motivation ( why the problem is challenging, what the changes, why we need to 
develop new models for the problem): 
Seizure Detection from EEG signals using wearable devices requires real-time processing that consumes 
minimal energy while maximizing accuracy. Traditional neural networks use the same computational cost, 
regardless of input complexity. Early exit architectures try to solve this issue by letting the network exit early if 
the network’s confidence is over a threshold. Existing approaches maintain constant width or increasing layer 
widths.  
Our project builds on these approaches by researching decreasing-width and adaptive-width early-exit 
architectures. Our hypothesis is that early layers need to capture and analyze more features in the input, which is 
the EEG patterns across channels and frequency bands in this case, while deeper layers focus on the specific 
discriminative features. This funnel type of design could reduce both memory usage and energy consumption 
compared to existing approaches to early exit architectures. 
This problem is challenging for multiple reasons. Seizure patterns in EEG signals vary across patients, 
so the model must detect them accurately while still benefiting from early exit. Because this is a medical use 
case, the models need a high recall score and provide well-calibrated confidence scores. Lastly, EEG datasets 
are small, making it easy for models with multiple exit stages to overfit. We hypothesize that width-adaptive 
architectures can achieve a better accuracy to efficiency ratio than existing approaches.  
 
d)​ (250 words max) (Rithwak) 
The (rough) methodology and plan for your project. Briefly explain the models you select and why you 
select. Give a structured plan with a set of incremental, implementable milestones and include a schedule 
for meeting your goals:  
Our plan is to evaluate 5 different model architectures.  
1.​ Deep Neural Network: Standard CNN without early exit, used as a baseline 
2.​ Constant-width early exit: Constant Layer width with exit heads at multiple depths 
3.​ Increasing-width early exit: Progressively wider layers with exit heads at multiple depths 
4.​ Decreasing-width early exit: Progressively narrower layers with exit heads at multiple depths 
5.​ Adaptive-width early exit: The network learns to remove irrelevant features dynamically. 
 
We chose five different models to evaluate because we wanted to compare our novel ideas with already existing 
approaches, with a normal CNN model as a baseline. Our models will be evaluated on accuracy, F1-score, 
exit-distribution, recall score, energy saved (FLOPs per inference), and confidence calibration. Our rough plan 
is to first focus on gathering accurate data and preprocessing it so that it is structured, formatted, normalized, 
and balanced. Then, we will implement the baseline deep neural network. Next, we will focus on making the 
existing approaches to the early exit architecture, which are the constant-width and increasing-width 
approaches. After that, we will implement the decreasing and adaptive-width architectures. Lastly, we will 
compare the 5 models side by side to analyze their performances and write our report. 
We plan to research existing approaches and evaluate multiple datasets first, and then work in parallel on all 5 
models and their visualizations/metrics, and lastly finalize the report alongside the results.

Datasets that you are planning to use for your experiment:  
Multiple EEG datasets (find a few and explain why they're good to use for this use case) (You need to

Dataset 1: CHB-MIT Scalp EEG Database: 
The CHB-MIT dataset contains EEG data from 23 different patients with drug-resistant epilepsy. It is accessible 
on PhysioNet and is commonly used for seizure detection research. It has been peer-reviewed and published by 
a trusted source (Boston Children’s Hospital), making it a reliable, well-established dataset. The recordings 
include 23-26 EEG channels sampled at 256 Hz, which is 983 hours of data and 198 documented seizures. This 
dataset provides us with detailed annotations, including seizure and non-seizure labels, timestamps, and 
information about seizure channels. 
Dataset 2: AES Seizure Prediction Dataset (Kaggle): 
The AES Seizure Prediction was recorded by the American Epilepsy Society for a machine learning 
competition. Its primary design was for seizure prediction, rather than simple detection. The dataset includes 
preprocessed EEG segments labeled as interictal (non-seizure) and preictal (pre-seizure). It contains EEG 
recording from 5 dogs and 2 humans, using 16 channels recorded at either 400 Hz or 5000 Hz. The dataset is 
very large, at over 100 GB, and includes thousands of 10-minute clips. While it is useful to develop and test the 
algorithm, it doesn’t have much useful clinical metadata compared to other datasets. 
Dataset 3: Siena Scalp EEG Dataset: 
The Siena Scalp EEG Dataset includes EEG recordings from 14 patients with various neurological conditions. It 
is also published on PhysioNet and follows standardized recording protocols from the University of Siena. The 
recordings each use 29-31 channels based on the 10-20 electrode placement system, sampled at 512 Hz. The 
dataset is vast, containing both short and long-term recordings, and its labels include seizure events, artifacts, 
and sleep stages, depending on the recording.

The resources needed to carry out your project: 
 
We will use Python because it has an extensive ecosystem of machine learning libraries and use cases. 
We will primarily use PyTorch to create and experiment with convolutional networks and other early-exit 
architecture models. If we ever need to, we can use TensorFlow and Keras for their deep learning capabilities. 
Numpy and Pandas will be an important part of our workflow, enabling efficient signal processing and dataset 
manipulation, both of which are needed to handle and analyze large volumes of data efficiently. We will use 
scikit-learn to evaluate our models with metrics such as accuracy, recall, and F1-score, and to assess the 
reliability of model predictions through confidence calibration. We will use MNE-Python for its preprocessing, 
filtering, segmentation, and visualization capabilities when working with EEG data.  
We plan to use Google Colab to train all our models because it offers free GPU access to everyone on 
the team and allows everyone to run the code without installation issues. Because it also connects to Google 
Drive, datasets, model checkpoints, and results can be easily shared and kept synchronized across the team, 
enabling efficient collaboration and reproducibility.  
Lastly, we will also use GitHub for version control. The repository will hold all source code, model 
definitions, training scripts, and Colab Notebooks, allowing all team members to efficiently track changes and 
avoid conflicts through branching and pull requests. Large datasets and model checkpoints will remain in 
Google Drive, while GitHub will maintain all tested code and the different experiment configurations. This will 
allow GitHub to serve as a professional repository where our research can be stored with different versions, and 
Google Drive to be used for temporary storage, running code, and other team needs.
